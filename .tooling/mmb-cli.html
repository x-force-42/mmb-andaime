Andaime MMB · ferramenta de inspeção

mmb — CLI do andaime

Janela única pra inspecionar o estado do andaime cross-repo: daemon, inboxes, repos, logs e journal. Read-only por padrão, com atrito intencional pra ações destrutivas.

.tooling/bin/mmb read-only por padrão bash puro sem deps novas

O que mmb é (e o que não é)

O andaime MMB orquestra trabalho entre três repos de produto (mmb-core, mmb-cockpit, mmb-aquarium) com Bash + tmux + mailbox em filesystem. mmb é a ferramenta unificada de inspeção desse estado em-voo.

Para que serve

Ver estado do commd, contar mensagens por inbox, listar postmortems, ler logs, rodar smoke tests, conferir se está pronto pra subir.

Para que não serve

Subir o andaime (up.sh), iniciar o daemon, fazer reset destrutivo (reset-all.sh), editar inbox/state/logs. Só lê — exceto daemon stop.

Início rápido

Os três comandos que você vai usar 80% do tempo

$ mmb # menu interativo — abre quando estiver perdido
$ mmb status     # resumo de uma tela
$ mmb doctor     # "estou pronto pra subir?"

Fluxo típico antes de subir o andaime

$ mmb preflight     # deps OK? gh autenticado? sem postmortems pendentes?
$ mmb smoke light    # canário rápido (~5s)
$ .tooling/bin/up.sh   # mmb NÃO sobe o andaime — use up.sh diretamente

Investigando um problema

$ mmb status                  # onde está o estranho?
$ mmb inbox dead              # olhar postmortems
$ mmb logs commd -n 100      # últimas 100 linhas do daemon
$ mmb logs core --follow    # seguir worker core ao vivo
$ mmb journal -n 30           # últimos eventos do journal

Flags globais

Reconhecidas em qualquer posição da linha de comando.

FlagEfeito
--yes, -yPula confirmações interativas (afeta apenas daemon stop hoje).
--help, -hMostra o usage e sai (0).
help (subcomando)Idem --help.

Menu interativo

Rodar mmb sem argumentos abre um menu numerado. Útil quando você não lembra exatamente o comando, ou quando quer navegar entre vários sem digitar.

=== MMB Andaime ===
commd:  RUNNING (pid=12345)
inbox:  master=0 core=3(1d) cockpit=0 aquarium=0
repos:  ⚠ task/foo em mmb-core

1) Status geral
2) Preflight
3) Smoke tests
4) Inbox / Dead letters
5) Daemons
6) Logs / Journal
7) Cleanup seguro
0) Sair
> _

Cada opção numerada abre um submenu ou executa o comando equivalente. Depois da execução, Enter volta pro menu raiz. b ou Enter vazio volta um nível. 0 sai.

Quando usarConveniência. Tudo aqui também tem um subcomando direto — se você já sabe o que quer, digite o subcomando.

Subcomandos

mmb status

mmb status

Resumo de uma tela do estado em-voo. Lê tudo, escreve nada.

O que mostra

  • commd: RUNNING, STALE (pid file aponta pra processo morto), ou STOPPED.
  • Inbox: por destinatário (master, core, cockpit, aquarium), contagem de mensagens em top-level, .processing, .done e .dead. Warning visual se há .dead.
  • Repos: branches task/* vivas em qualquer dos 3 repos, mais aviso se há mudanças não-commitadas.
  • Journal: último evento de logs/journal.jsonl (formatado com jq se disponível).

Exemplo

$ mmb status

=== MMB Andaime — status ===
  commd:       RUNNING (pid=79605)

Inbox:
  inbox/master:      top=0 processing=0 done=12 dead=0
  inbox/core:        top=1 processing=0 done=8  dead=2 ⚠
  inbox/cockpit:     top=0 processing=0 done=0  dead=0
  inbox/aquarium:    top=0 processing=0 done=0  dead=0

Repos:
  ⚠ task/cleanup-scripts in mmb-core

Journal (último evento):
  {"ts":"2026-05-15T18:42:01Z","event":"commd-done","dest":"core",...}

Exit code: sempre 0.

mmb preflight

mmb preflight

Diagnóstico antes de subir o andaime. Não age — só reporta.

O que verifica

  1. Deps críticas: inotifywait, gh, tmux, git, claude, bash.
  2. Deps opcionais: jq, python3.
  3. Autenticação no GitHub (gh auth status).
  4. Single-instance do commd (inspeciona o pid file e o lock, sem segurar).
  5. Postmortems pendentes em inbox/*/.dead/.
  6. Branches task/* e git status sujo em qualquer dos 3 repos.

Exit codes

0✓ pronto pra subir
1⚠ pode subir, com cuidado (postmortems, branches task/* vivas, deps opcionais ausentes)
2✗ não suba (dep crítica ausente, gh não autenticado)

Sem redepreflight não faz git fetch nem chama API do GitHub além do que gh auth status precisa localmente.

mmb smoke <modo>

mmb smoke {light|medium|stress|hardening}

Wrapper fino sobre .tooling/bin/smoke.sh. Roda em sandbox isolado em /tmp/mmb-smoke-XXXXXX — não toca o andaime real.

ModoDuraçãoPara que serve
light~5sCanário rápido (C1-C6) com mock do claude.
medium~22sFalhas controladas + recovery.
stress~17sVolume e concorrência.
hardening~28slight + medium (CI / validação pré-push).

Comm/aquario fora do escopoOs modos comm e aquario do smoke.sh exigem commd vivo + claude real. Use .tooling/bin/smoke.sh comm diretamente.

Exemplo

$ mmb smoke hardening
=== HARDENING: light + medium ===
...
=== HARDENING PASS ===

mmb inbox

mmb inbox [dead|processing]

Lista mensagens nos 4 destinatários, agrupadas. Sem argumento mostra top-level (pendentes).

VarianteO que lista
mmb inboxPendentes em inbox/<dest>/*.md.
mmb inbox processingEm voo (worker as alegou) em .processing/.
mmb inbox deadPostmortems (worker timeout / non-zero exit) em .dead/.

Formato por linha

  core      2026-05-15T14:23:01Z  master->core  briefing     cleanup-scripts

(vazio) aparece para cada destinatário sem mensagens.

mmb daemon

mmb daemon {status|stop|start}

Wraps finos sobre commd.sh, com confirmação e mensagens orientadas.

mmb daemon status

Mostra RUNNING (pid=N), STALE (pid=N from file, process dead), ou STOPPED.

mmb daemon stop

Único subcomando do mmb que envia sinal a processo. Pede confirmação:

Stop commd daemon? [y/N] _

Pule a confirmação com --yes:

$ mmb daemon stop --yes

Stdin não-ttySem --yes e sem tty (rodando dentro de pipe ou script), o comando aborta com mensagem clara. Isso é proposital — confirmação que não pode ser lida do tty é confirmação fake.

mmb daemon start

Não inicia nada. Imprime instruções apontando pra up.sh (setup completo) ou commd.sh fg (debug em foreground). Sai com 0.

mmb logs

mmb logs [target] [--follow] [-n N]

Tail de logs do daemon e dos workers.

TargetArquivo
commdlogs/commd.log
corelogs/workers/core.log
cockpitlogs/workers/cockpit.log
aquariumlogs/workers/aquarium.log
masternão existe — sai com 2 e lista alternativas.

Opções

  • -n N — últimas N linhas (default: 50).
  • --follow, -f — segue o arquivo (tail -F). Sai limpo com Ctrl+C.

Sem target

Se rodado sem target em terminal interativo, apresenta um menu numerado 1) commd 2) core 3) cockpit 4) aquarium.

Exemplos

$ mmb logs commd -n 100
$ mmb logs core --follow
$ mmb logs                # menu numerado

mmb journal

mmb journal [--follow] [-n N]

Tail de logs/journal.jsonl — o log estruturado de eventos do commd (dispatch, claim, done, dead, worker-timeout, ...). Default: últimas 20 linhas.

Se jq estiver instalado, formata cada linha com jq -c '.'. Senão, mostra cru.

Exemplo

$ mmb journal --follow
{"ts":"2026-05-15T18:42:01Z","event":"commd-dispatch","dest":"core","file":"...","pid":79605}
{"ts":"2026-05-15T18:42:01Z","event":"commd-claim","dest":"core","file":"...","pid":79605}
{"ts":"2026-05-15T18:42:14Z","event":"commd-done","dest":"core","file":"...","pid":79605}
{"ts":"2026-05-15T18:42:14Z","event":"commd-worker-done","dest":"core",...}

mmb doctor

mmb doctor

Composição: preflight + smoke light + sumário final.

Se o preflight der crítico, pula o smoke (não dá pra confiar). Senão, roda smoke light.

Veredicto

✓ prontopreflight e smoke passaram.
⚠ subir com cuidadoalgum aviso não-bloqueante.
✗ não suba aindafalha crítica em algum check.

Exit code: max(preflight, smoke).

mmb cleanup

mmb cleanup

Puramente informativo. Imprime as opções de cleanup disponíveis e como invocá-las manualmente. Não executa nada.

  • Reset total via reset-all.sh (com --dry-run, interativo, ou --yes).
  • Arquivar .dead/ sem reset total.
  • Truncar journal.jsonl.

Por quê informativoCleanup destrutivo é um momento que merece atenção manual. mmb mostra o cardápio, você escolhe e digita.

mmb reset

mmb reset

Stub informativo — aponta pra .tooling/bin/reset-all.sh. Sai com 0.

Atrito intencionalmmb nunca invoca reset-all.sh, nem com flag. Reset total fecha PRs, deleta branches em todos os repos, arquiva mensageria — quer digitar o caminho completo.

mmb up

mmb up

Stub informativo — aponta pra .tooling/bin/up.sh. Sai com 0.

$ .tooling/bin/up.sh                       # normal (Opus tudo)
$ MMB_MODE=fast .tooling/bin/up.sh         # smoke
$ .tooling/bin/up.sh --no-master-claude    # master em sessão externa

Por quê não wrapeiaSetup do tmux + spawn de sessões claude tem variações e env vars que mudam frequentemente. O ponto de entrada canônico é up.sh; o mmb não adiciona valor encapsulando essa chamada.

Princípios de design

  1. Inspeção, não gestão. mmb não substitui up.sh. Setup de tmux + daemons continua via up.sh.
  2. Read-only por padrão. Único subcomando que envia sinal é daemon stop (com confirmação).
  3. Não duplica lógica. mmb smoke light é exec smoke.sh light. mmb adiciona UX (cor, contexto, confirmação), não comportamento.
  4. Não invoca reset-all.sh, nem com flag. Atrito intencional.
  5. Não escreve em inbox/, state/, intents/, nem trunca logs.
  6. Sem TUI fullscreen. Menu numerado com read. Sem navegação por setas, sem redraw complexo.
  7. Bash puro. Cores via ANSI; nada de gum/whiptail/dialog/fzf.

Exit codes

CódigoSignificado
0Sucesso (ou só informativo).
1Aviso (preflight com warnings; daemon stop abortado).
2Erro de uso (subcomando desconhecido, argumento inválido, andaime nunca inicializado) ou falha crítica de preflight.
outrosRepassados do comando subjacente (smoke, commd.sh).

Cor & NO_COLOR

Cores ANSI são automaticamente desabilitadas quando:

  • A saída padrão não é um tty (pipe, redirect, captura).
  • A variável de ambiente NO_COLOR está definida (no-color.org).
$ NO_COLOR=1 mmb status
